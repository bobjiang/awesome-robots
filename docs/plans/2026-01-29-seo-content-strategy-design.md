## Section 2: Universal Content Template & Structure

Every guide follows this proven 8-section structure (3000-5000 words total):

### 1. Hero Section (200-300 words)
- Compelling hook addressing reader's pain point
- Clear value proposition (what they'll learn)
- Key statistics or market context (e.g., "105 robots analyzed", "prices from $1,600 to $250,000")
- Table of contents for easy navigation
- Reading time estimate

### 2. Market Context (400-500 words)
- Current state of the category (2026 landscape)
- Price ranges and availability overview
- Major manufacturers and market trends
- Technology breakthroughs relevant to buyers
- Market growth statistics and projections

### 3. Decision Framework (600-800 words)
- 5-7 key buying criteria with detailed explanations
  - Budget and total cost of ownership
  - Use case alignment and application requirements
  - Technical specifications (DOF, payload, battery, sensors)
  - Software ecosystem (ROS support, SDK quality, AI integration)
  - Vendor reliability and support
  - Upgrade path and future-proofing
  - Community and documentation
- Budget considerations and ROI factors
- Technical requirements checklist
- Use-case alignment questions
- Common pitfalls to avoid

### 4. Detailed Product Comparisons (1200-1500 words)
- Comprehensive comparison tables (top 8-12 robots from catalog)
- Category-specific analysis:
  - Humanoids: dexterity, bipedal stability, manipulation capabilities
  - Quadrupeds: terrain capability, payload capacity, weatherproofing
- Price-to-performance analysis
- Availability and lead times
- Warranty and support comparison
- Feature matrices organized by buyer priority

### 5. Use-Case Recommendations (500-700 words)
- 3-5 specific buyer personas or applications
  - Example: "Best for warehouse automation", "Best for outdoor research", "Best for education"
- "Best for X" recommendations with detailed rationale
- Budget tiers with recommendations:
  - Entry-level (<$20k)
  - Mid-range ($20k-$50k)
  - Premium ($50k-$100k)
  - Enterprise ($100k+)
- Direct links to robot detail pages and quote forms
- Side-by-side comparisons of top picks

### 6. Implementation Guidance (300-400 words)
- Getting started checklist
- Integration considerations (existing systems, infrastructure)
- Training and support resources
- Timeline expectations (procurement, setup, deployment)
- Safety and compliance requirements
- Maintenance planning

### 7. Comprehensive FAQ (400-600 words)
- 8-12 frequently asked questions with detailed answers
- FAQ schema markup for rich snippets in search results
- Covers:
  - Technical questions (specs, compatibility, integration)
  - Pricing questions (hidden costs, financing, ROI)
  - Support questions (warranty, training, spare parts)
  - Deployment questions (timeline, setup, safety)
- Strategic keyword targeting in questions

### 8. Conclusion & CTA (200-300 words)
- Summary of key takeaways (3-5 bullet points)
- Next steps for different buyer types:
  - Researchers: "Request university pricing"
  - Enterprises: "Schedule consultation"
  - Hobbyists: "Browse entry-level robots"
- Newsletter signup (weekly updates on new robots)
- Quote request forms (contextual to guide topic)
- Consultation booking for high-value prospects
- Internal links to related guides and robot pages

## Section 6: Content Production Workflow & Template System

### Content Template Files

**Location:** `content/templates/`

**Template Files to Create:**

1. **`buying-guide-template.md`** - Standard buying guide structure
2. **`deep-dive-guide-template.md`** - Comprehensive category overview structure
3. **`comparison-guide-template.md`** - Framework for comparative analysis pieces

### Template Structure

Each template includes:

**Frontmatter Schema:**
```markdown
---
title: "[Guide Title]"
slug: "[url-slug]"
author: "bob-jiang"
date: "YYYY-MM-DD"
category: "guides"
tags: ["tag1", "tag2", "tag3"]
excerpt: "[150-160 character summary]"
featured: true
published: false  # Set to true when ready
contentType: "guide"
wordCount: 0  # Update when complete
readingTime: 0  # Update when complete
seo:
  title: "[60 chars max - includes primary keyword]"
  description: "[155-160 chars - compelling, includes CTA]"
  keywords: ["primary keyword", "secondary keyword 1", "secondary keyword 2", ...]
---
```

**Section Headers with Word Count Targets:**
```markdown
## [Section Name] (TARGET: XXX words)

[Placeholder text explaining what goes in this section]

**Key points to cover:**
- Point 1
- Point 2
- Point 3

**Internal links to add:**
- [ ] Link to related guide
- [ ] Link to robot detail page
- [ ] Link to category page
```

**Quality Checklist (at end of template):**
```markdown
## Pre-Publish Checklist

### Content Quality
- [ ] Word count: 3000-5000 words
- [ ] Reading level: Grade 10-12 (check readability score)
- [ ] All sections complete
- [ ] Comparison tables included
- [ ] FAQ section (8-12 questions)

### SEO Optimization
- [ ] Primary keyword in title, H1, first paragraph
- [ ] Secondary keywords distributed naturally
- [ ] Meta description compelling and includes CTA
- [ ] Alt text for all images
- [ ] Internal links: 15-20 relevant links
- [ ] External links: 3-5 authoritative sources

### Technical
- [ ] Schema markup: Article, FAQ, Breadcrumb
- [ ] OG image created/generated
- [ ] All links tested (no 404s)
- [ ] Builds successfully locally
- [ ] Mobile responsive check

### Editorial
- [ ] Fact-checked (specs, pricing, availability)
- [ ] Proofread for grammar/spelling
- [ ] CTAs clear and contextual
- [ ] Legal review if needed (claims, comparisons)
```

### Production Workflow

**Per Guide Timeline: 16-25 hours total**

#### Phase 1: Research (4-6 hours)

**Keyword Research:**
- Use Ahrefs/SEMrush for search volume and competition
- Identify primary keyword (1000-5000 searches/month)
- Find 3-5 secondary keywords (500-2000 searches/month)
- Compile 10-15 long-tail keywords (100-500 searches/month)

**Competitor Analysis:**
- Identify top 5 ranking articles for primary keyword
- Analyze their structure, length, content gaps
- Note what they do well, what's missing
- Identify differentiation opportunities

**Data Extraction:**
- Query robots.json for relevant products
- Extract specifications for comparison tables
- Verify pricing and availability
- Note unique features for highlighting

**External Research:**
- Gather manufacturer whitepapers and technical docs
- Find relevant research papers (for credibility)
- Collect real-world deployment case studies
- Screenshot/save external images needed

#### Phase 2: Outline Creation (1-2 hours)

- Copy appropriate template to `content/blog/guides/[slug].md`
- Customize template for specific guide topic
- Define featured products list (8-20 robots)
- Map out comparison tables and data visualizations
- Outline use-case scenarios and buyer personas
- Plan FAQ questions (target featured snippets)
- Identify internal linking opportunities

#### Phase 3: Writing (8-12 hours)

**Hero Section (30 min):**
- Write compelling hook (problem/opportunity)
- Craft value proposition
- Add key statistics from research
- Create table of contents

**Market Context (1 hour):**
- Summarize 2026 landscape
- Price ranges and availability
- Major trends and manufacturers
- Technology breakthroughs

**Decision Framework (1.5 hours):**
- Write 5-7 buying criteria sections
- Explain each criterion in detail
- Add budget and ROI considerations
- Create technical requirements checklist
- List common pitfalls

**Product Comparisons (3-4 hours):**
- Build comparison tables from robots.json data
- Write analysis of each comparison dimension

- Note availability and lead times
- Compare warranty and support

**Use-Case Recommendations (1.5 hours):**
- Define 3-5 buyer personas

**Implementation Guidance (1 hour):**
- Create getting started checklist

**FAQ (1.5 hours):**
- Write 8-12 FAQ pairs

**Conclusion & CTA (30 min):**
- Summarize key takeaways
- Write next steps for different buyer types
- Add newsletter signup, quote requests

#### Phase 4: Enhancement (2-3 hours)

**Visual Assets:**
- Create/source featured image (1200x630px)
- Build custom comparison charts

- Create data visualizations
- Optimize all images (WebP format, compression)

**Readability Optimization:**
- Add descriptive headers and subheaders
- Use bullet points and numbered lists

**Schema Markup:**
- Add Article schema (auto-generated by Velite)

**Meta Optimization:**
- Write compelling meta description (155-160 chars)

#### Phase 5: Review & Publishing (1-2 hours)

- Deploy to production
- Submit to Google Search Console

### Parallelization Strategy

**For 9 Guides Total (144-225 hours):**

**Option 1 - Sequential (1 person):**
- 1 guide per week = 9 weeks total
- Allows for learning and refinement
- Lower risk, higher quality

**Analysis:**
1. Compare month-over-month metrics for each guide
2. Identify underperforming guides (low traffic, high bounce rate, low conversions)
3. Identify overperforming guides (learn what's working)
4. Check keyword ranking changes (wins and losses)
5. Review backlink growth (new links, lost links)
6. Analyze user behavior (scroll depth, click patterns, exit pages)

**Optimization Priorities:**
1. Underperforming guides: Diagnose issue (poor keyword targeting? Weak content? Low rankings?)

3. Content gaps: Identify keywords we should target but aren't
4. Technical SEO: Fix any crawl errors, schema issues, broken links

**Documentation:**

- Plan next month's optimization tasks
